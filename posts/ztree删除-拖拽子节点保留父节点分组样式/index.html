zTree 删除/拖拽子节点保留父节点分组样式

2019-04-12
阅读时长: 3 分钟

最近需要在项目中实现报表的自定义设置功能，即用户可以针对报表新建自定义分组，分组间可以互相嵌套，分组及分组内的报表需要支持拖拽排序、编辑、删除……相信听到这里，你大概明白我要实现一个什么样的功能了。不错，我要实现一个集美观、功能于一身的树形菜单。本着“不要重复制造轮子”的原则，我在考察了 JQuery EasyUI、layui、Bootstrap、Kendo UI 等不同框架提供的“树形菜单”组件以后，最终选择了zTree这样一个插件，虽然这个官网看上去相当复古，虽然最终的成品依然被同事吐槽丑，可它的确完美得实现了我想要的功能，是当之无愧的“树形菜单”王者。

zTree 的 API 相当复杂，尤其是属性和事件的种类，简直叫一个繁杂，这是大部分基于 jQuery 插件的一个特点。不过 zTree 的使用还是比较简单的，我们只需要提供一个 DOM 节点，一份 JSON 数据，zTree 就可以帮我们在界面上渲染出一个完整的树形菜单：

1var data = res.Data;
2var zNodes = JSON.parse(data.TreeData);
3$.fn.zTree.init($("#reportTree"), setting, zNodes);

zTree 的节点是由 JSON 结构来定义的，其基本结构是{name:“节点名称”,children:[]}，父子节点采用相同的结构相互嵌套。例如，下面是博主所使用的数据结构：

参考官网上的 DEMO，我们基本上就可以快速上手 zTree，博主这里就是结合了节点的编辑、拖拽这两个功能。不过，按照官网上的 DEMO 会存在两个 Bug，与我们实际的期望有所不同，**其一，是当一个分组下的子节点被全部删除后，这个分组的图标会变成一个子节点的图标；其二，是当个一个分组下的节点被全部拖拽到分组以外的地方，这个分组的图标会变成一个子节点的图标。**这两个 Bug 是由测试小姐姐们发现的，zTree 是我引入到项目中来的，这个 Bug 哪怕跪着都要改完，说多了都是泪啊，下面给出解决方案：

 1function onRemove(e, treeId, treeNode) {
 2    var zTree = $.fn.zTree.getZTreeObj(reportTreeId);
 3    var root = zTree.getNodes()[0];
 4    if (treeNode.isParent) {
 5        reports = GetReportsByNode(treeNode)
 6        var parentNode = treeNode.getParentNode();
 7        if (parentNode != null && (parentNode.children == null || parentNode.children.length == 0)) {
 8            parentNode.isParent = true;
 9            parentNode.isOpen = true;
10            zTree.updateNode(parentNode);
11        }
12    }
13}
14
15var emptyNode;
16function beforeDrop(treeId, treeNodes, targetNode, moveType, isCopy) {
17    var zTree = $.fn.zTree.getZTreeObj(reportTreeId);
18    for (var i = 0; i < treeNodes.length; i++) {
19        var treeNode = treeNodes[i];
20        var parentNode = treeNode.getParentNode();
21        if (parentNode != null && (parentNode.children == null || parentNode.children.filter(function (s) { return s.name != treeNode.name; }).length == 0)) {
22            emptyNode = parentNode;
23            break;
24        }
25    }
26
27    return true;
28}
29
30function onDrop(event, treeId, treeNodes, targetNode, moveType, isCopy) {
31    var zTree = $.fn.zTree.getZTreeObj(reportTreeId);
32    if (emptyNode != null) {
33        emptyNode.isOpen = true; emptyNode.isParent = true;
34        zTree.updateNode(emptyNode);
35        emptyNode = null;
36    }
37}

OK，实际项目中可能需要存储这个树形结构，因为你能想象，用户编辑完这样一个“个性化”的设置以后，我们还要根据这个设置来加载树形菜单，以达到个性化的目的。那么，怎么获得这个树形结构呢，理论上我们只需要通过 zTree.getNodes()方法获得整个树的节点信息，然后将其序列化为 JSON 即可，可实际上 zTree 会在树上附加“冗余”信息，所以，博主的做法是，通过递归来遍历整个树的节点，获取其中的关键信息，这里以 name 字段为例：

 1function GetTreeData(zTree) {
 2    var data = [];
 3    for (var i = 0; i < zTree.length; i++) {
 4        var treeNode = zTree[i];
 5        if (!treeNode.isParent) {
 6            var obj = new Object();
 7            obj.name = treeNode.name;
 8            data.push(obj)
 9        } else {
10            var obj = new Object();
11            obj.name = treeNode.name;
12            obj.children = GetTreeData(treeNode.children)
13            data.push(obj)
14        }
15    }
16
17    return data;
18}
